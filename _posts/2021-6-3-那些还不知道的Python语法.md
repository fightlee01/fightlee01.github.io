# 那些还不知道的Python语法
## __getitem__方法
__getitem__方法是一个魔改方法，它可以改变类的实例访问的方式。例如demo为一个类的实例对象，当使用demo[item]时调用该方法。
```python
class Demo:
    def __init__(self, name, age):
        self.name = name
        self.age = age
    
    def __getitem__(self, item):
        return item


demo = Demo('lee', 22)
print(demo[123])#123

print(demo['fight'])#fight
```

# 图像分类上的对抗鲁棒性检测
## 摘要
深度神经网络容易受到对抗样本的影响，这成为深度学习发展中最重要的研究问题之一。虽然近年来已经做出了很多努力，但对对抗性攻击和防御算法进行正确和完整的评估具有重要意义。在本文中，我们建立了一个全面、严格且连贯的基准来评估图像分类任务的对抗性鲁棒性。在简要回顾了大量具有代表性的攻击和防御方法后，我们以两条鲁棒性曲线作为公正的评估标准进行了大规模实验，以充分了解这些方法的性能。基于评估结果，我们得出了几个可以为未来研究提供见解的重要发现，包括：1）模型之间的相对稳健性可以在不同的攻击配置下发生变化，因此鼓励采用稳健性曲线来评估对抗性稳健性； 2）作为最有效的防御技术之一，对抗训练可以泛化到不同的威胁模型； 3）基于随机化的防御对基于查询的黑盒攻击更健壮。 
## 1 引言
深度学习 (DL) 模型容易受到恶意生成的对抗样本的影响，从而导致错误预测。 由于深度学习模型已集成到各种对安全敏感的应用程序（例如，自动驾驶、医疗保健和金融）中，随着大量对抗性攻击和防御方法的提出，对抗鲁棒性问题的研究引起了越来越多的关注。 因此，对这些方法进行正确和严格的评估以了解它们的优缺点、比较它们的性能并为构建新方法提供见解至关重要。  
对抗鲁棒性的研究面临着攻击和防御之间的“军备竞赛”，即为了防止现有攻击提出一种防御方法很快又被新的攻击规避，反之亦然。 例如，有人提出防御蒸馏来提高对抗性的鲁棒性，但后来证明对强大的攻击无效。许多方法引入了通过混淆梯度来构建稳健模型，这些方法可以被自适应梯度击败。因此，了解它们的影响、确定真正的进展并推进该领域尤其具有挑战性。  
此外，当前的攻击和防御往往评估不完整。首先，大多数防御仅针对有限威胁模型下的一小部分攻击进行测试，并且许多攻击是针对少数模型或防御进行评估的。 其次，稳健性评估指标过于简单，无法显示这些方法的性能。 对于给定的扰动预算，防御攻击的准确性和对抗性扰动的最小距离被用作主要评估指标，这通常不足以完全表征攻击和防御的行为。 因此，不完整的评估无法全面了解这些方法的优势和局限性。  
在本文中，我们建立了一个全面、严格和连贯的基准来评估对抗鲁棒性，它可以详细了解现有方法在不同场景下的效果，以期促进未来的研究。特别是，我们专注于 $\mathcal{l}_p$规范威胁模型下图像分类器的鲁棒性，其中已经投入了大量工作。我们结合了许多典型的和最先进的攻击和防御方法进行鲁棒性评估，包括 15 种攻击方法和16种防御模型——CIFAR-10上的8种和ImageNet上的8种。为了充分展示这些方法的性能，我们采用两条互补的鲁棒曲线作为主要评估指标来呈现结果。然后，我们在完整威胁模型下进行了大规模的攻击和防御方法交叉评估实验，包括1）无针对性和有针对性的攻击；2) $l_\infin$ 和$l_2$范数攻击；3) 白盒、基于传输、基于分数和基于决策的攻击。  
通过分析定量结果，我们有一些重要的发现。首先，在不同的扰动预算或攻击迭代下，防御之间的相对鲁棒性可能不同。因此，很难得出这样的结论：通过使用特定的配置，一种防御比另一种防御更能抵御攻击。但是，这在以前的作品中很常见。 其次，虽然已经提出了各种防御技术，但最强大的防御仍然是对抗训练的模型。它们的鲁棒性也可以推广到其他威胁模型，在这些模型下，它们没有被训练为鲁棒性。第三，基于随机化的防御通常比基于查询的黑盒攻击更健壮。更多讨论可以在5.3节中找到。  
我们开发了一个名为RealSafe的新的对抗性健壮性平台来进行所有的评估实验，因为现有的平台（如CleverHans和Foolbox）不能完全支持我们的评估（见附录a）。我们希望我们的平台能够不断地融合和评价更多的方法，为以后的工作提供帮助。  
## 2 威胁模型
精确定义威胁模型是进行对抗鲁棒性评估的基础。根据该理论，威胁模型规定了对抗的目标、能力和知识，在这些目标、能力和知识下进行攻击并建立起稳健的防御体系。我们首先定义符号，然后说明威胁模型的三个方面。  
一个分类器可以表示为：$\mathcal{C}(x):\mathcal{X}\to\mathcal{Y}$，其中，输入是$x\in\mathcal{X}\subset\R^d$，输出$\mathcal{Y}=\{1,2,...,L\}$，L为类别数目。$y$表示输入$x$的真实标签，$x^{adv}$表示$x$的对抗样本。
### 2.1 攻击目标
攻击者有不同的目标生成对抗样本。本文研究非目标和目标对抗样本。非目标对抗样本的目标是使分类器误分类，即$\mathcal{C}(x^{adv})\ \ne\ y$。目标对抗样本是使分类器误分类为攻击者渴望的类别，即$\mathcal{C}(x^{adv})\ \ne \ y^*$，$y^*$是目标类别。
### 2.2 攻击能力
由于对抗样本通常被假设为与人眼无法区分的对应原始样本，因此攻击者只能对输入进行很小的更改。尽管也存在其他威胁模型，但在本文中我们研究了定义明确且广泛使用的$\mathcal{l}_p$范数威胁模型。在$\mathcal{l}_p$范数威胁模型下，允许攻击者在原始输入中添加由$\mathcal{l}_p$范数衡量的小扰动。具体来说，我们考虑$\mathcal{l}_{\infty}$和$\mathcal{l}_2$范数。   
为了实现攻击目标，可以采用两种小扰动策略来生成对抗样本。第一种方法试图构建一个满足$||x^{adv}-x||_p\ \le\ {\epsilon}$的对抗样本$x^{adv}$误导模型，其中${\epsilon}$为扰动预算。这可以通过一个约束优化问题实现。例如，攻击者可以通过最大化一个很小区域内的损失函数$\mathcal{J}$（例如交叉熵函数）来获得非目标对抗样本。$$x^{adv}=argmax_{x^{'}:||x^{'}-x||_p\le\epsilon}\mathcal{J}(x^{'},y).\tag{1}$$我们称之为受限扰动的对抗样本。第二个策略是通过找到最小化扰动xxx来生成对抗样本。我们称它为优化扰动对抗样本。然而，精确求解（1）或（2）是困难的，因此，提出各种攻击方法来获得近似解。
